# 공간 네비게이션 (Spatial Navigation) 기획 명세

## 개요

현재 독립된 페이지였던 로그인·공개 블로그를 **중앙 허브 화면**을 기준으로 상하좌우로 슬라이딩하는 공간(spatial) 구조로 재편한다.
키보드·마우스 휠·터치 스와이프 모두 지원하며, 전환은 CSS `transform: translateX/Y`로 하드웨어 가속 슬라이드.

---

## 공간 배치도

```
                    ┌─────────────────┐
                    │   [위] 예비 슬롯  │  (추후 확장)
                    │   (미구현)       │
                    └────────┬────────┘
                             │ ↑ 위로 스와이프
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         │  [좌] 예비 슬롯    │   [중앙] 메인 허브 │──→──  [우] 로그인
         │  (미구현)         │   (파티클 스플래시) │
         │                   │                   │
         └───────────────────┼───────────────────┘
                             │ ↓ 아래로 스와이프
                    ┌────────┴────────┐
                    │  [아래] 공개 블로그│
                    │  (Brunch 스타일) │
                    └─────────────────┘
```

| 방향 | 화면 | 설명 |
|------|------|------|
| 중앙 | **메인 허브** | 현재 Login.tsx의 파티클 애니메이션 영역을 전체 화면으로 확장. 로그인 폼 제거. |
| → 우 | **로그인** | 기존 로그인 폼. 중앙에서 우로 슬라이드하면 진입. |
| ↓ 아래 | **공개 블로그** | Brunch 스타일 감성 블로그 피드. 비인증 접근 가능. |
| ↑ 위 | (예비) | 추후 확장용 슬롯. |
| ← 좌 | (예비) | 추후 확장용 슬롯. |

## 1. 중앙 허브 (Splash Hub)

### 역할
- 앱 진입점. 브랜드 첫인상.
- 방향 힌트(Breathing Indicator)로 이동 가능 방향 안내.

### 레이아웃
```
┌─────────────────────────────────────────┐
│                                          │
│         [파티클 애니메이션 전체화면]         │
│              U.T Lab                     │  ← 기존 ParticleCanvas 유지
│                                          │
│                   ↓ (브리딩 인디케이터)      │  ← 블로그 방향 암시
│                                          │
│                              → (브리딩)  │  ← 로그인 방향 암시
│                                          │
│  "공개 블로그 보기" 텍스트 (↓ 아래 힌트)    │
└─────────────────────────────────────────┘
```

### 브리딩 인디케이터 (Breathing Indicator) 디자인

```
각 방향 끝에 반투명 chevron 아이콘이 opacity 0.15 ~ 0.40을
2.4초 주기로 페이드인·아웃 반복 (ease-in-out, 루프).

우측 인디케이터:  ›  (ChevronRight)
하단 인디케이터:  ⌄  (ChevronDown)

위치: 화면 가장자리에서 안쪽으로 32px
색상: rgba(particle_rgb, 0.25)  →  파티클 색상 기반
크기: 20px (모바일 24px)
애니메이션: @keyframes breathe
  0%   { opacity: 0.12; transform: translateX(0) }
  50%  { opacity: 0.38; transform: translateX(4px) }  ← 우측은 X, 하단은 Y
  100% { opacity: 0.12; transform: translateX(0) }
```

## 2. 우측 → 로그인

### 기존 Login.tsx 재활용
- 현재 로그인 폼(우측 패널)만 유지.
- 파티클 영역 제거 (중앙 허브가 담당).
- 배경: 중앙 허브와 연결되는 어두운 그라디언트 유지.

### 닫기 / 돌아가기
- 좌측 스와이프 또는 `Escape` 키 → 중앙으로 복귀.
- 우측 상단 `← 뒤로` 브리딩 인디케이터 (ChevronLeft).

## 3. 아래 ↓ 공개 블로그 (Brunch 스타일)

### 컨셉
```
브런치(brunch.co.kr) 레퍼런스:
- 큰 커버 이미지 중심
- 넓은 여백, 깔끔한 타이포그래피
- 카드 호버 시 제목이 위로 슬라이드하며 나타남
- 필자명 + 날짜 서브텍스트
- 다크/라이트 모드에서도 감성 유지
```

### 레이아웃 구조
```
┌─────────────────────────────────────────────┐
│  ↑ (브리딩 인디케이터 — 위로 돌아가기)           │
│                                              │
│  U.T Lab  ·  글                              │  ← 미니멀 헤더
│  ─────────────────────────────────────────── │
│                                              │
│  ┌─────────────────────────────────────────┐ │
│  │  [HERO CARD — 최신 글]                   │ │  ← 전폭 히어로
│  │  커버이미지 (16:9, object-fit: cover)     │ │
│  │                                          │ │
│  │  제목 (28px, 감성 폰트)                   │ │
│  │  발행일 · 읽기 시간                        │ │
│  └─────────────────────────────────────────┘ │
│                                              │
│  ┌───────────┐  ┌───────────┐  ┌──────────┐  │  ← 3열 그리드 (md 이상)
│  │ [썸네일]  │  │ [썸네일]  │  │ [썸네일] │  │     2열 (sm), 1열 (xs)
│  │           │  │           │  │          │  │
│  │ 제목      │  │ 제목      │  │ 제목     │  │
│  │ 날짜 · 분 │  │ 날짜 · 분 │  │ 날짜 ·분 │  │
│  └───────────┘  └───────────┘  └──────────┘  │

│                                              │
└─────────────────────────────────────────────┘
```

### 카드 상세 디자인

**Hero Card (최신 글)**
```
- aspect-ratio: 21/9 (와이드)
- 이미지 위에 하단 그라디언트 오버레이 (투명 → 검정 50%)
- 제목: 흰색, 2.2rem, font-weight: 700
- 발행일: 흰색 70%, 0.85rem
- 호버: 이미지 scale(1.02) 2s ease
```

**일반 카드**
```
- aspect-ratio: 4/3 (이미지)
- 이미지 없는 경우: 강조색 그라디언트 배경 + 첫 문장 발췌
- 제목: 1rem, font-weight: 600, 2줄 클램프
- 발췌문: 0.85rem, zinc-500, 3줄 클램프
- 태그: 작은 pill 뱃지
- 호버: translateY(-4px), shadow 강화
```

**커버 이미지 없는 글 폴백**
```
배경: linear-gradient(135deg, accent/20, accent/5)
중앙: 첫 글자 대문자 이니셜 (4rem, accent 색)
```

## 4. 전환 애니메이션

### CSS Transform 기반 슬라이드

```
레이아웃:
  각 "화면"은 100vw × 100vh 고정 크기의 패널.
  전체 래퍼가 translate로 이동 (개별 페이지 이동 X).

래퍼 구조 (논리):
  ┌──────────────────────────────────────────────────────┐
  │          [위 슬롯 100vw×100vh]                        │
  │  [좌 슬롯]  [중앙 허브]  [우: 로그인]                  │
  │          [아래: 블로그]                               │
  └──────────────────────────────────────────────────────┘

  이동:
    우로 → wrapper.style.transform = "translateX(-100vw)"
    아래로 → wrapper.style.transform = "translateY(-100vh)"
    복귀 → wrapper.style.transform = "translate(0, 0)"

전환 CSS:
  transition: transform 0.65s cubic-bezier(0.76, 0, 0.24, 1)
  (느리게 시작, 가속 후 부드럽게 감속 — iOS 네이티브 느낌)
```

### 입력 방식별 트리거

| 입력 | 조건 | 동작 |
|------|------|------|
| 터치 스와이프 | 방향 이동 > 60px, 세로/가로 비율 < 0.5 | 해당 방향 화면으로 전환 |
| 마우스 휠 | deltaY > 80 (아래 방향) | 블로그로 전환 (중앙에서만) |
| 키보드 Arrow | ArrowRight / ArrowDown | 해당 방향 전환 |
| 키보드 Escape | 중앙 이외 위치 | 중앙으로 복귀 |
| 브리딩 인디케이터 클릭 | — | 해당 방향 전환 |

### 스냅(snap) 동작
- 스와이프 중 50% 초과 시 자동 완주
- 50% 미만 시 원래 위치로 복귀 (스프링 백)
- 전환 중 추가 입력 무시 (isTransitioning 플래그)

## 7. 접근성 / 엣지 케이스

- **로그인 직접 URL 접근** (`/login`): 공간 네비게이션 없이 단독 렌더링 유지.
- **공개 블로그 직접 URL 접근** (`/public/blog`): 단독 렌더링 유지.
- **prefers-reduced-motion**: 슬라이드 전환을 `opacity` 크로스페이드로 대체. 브리딩 애니메이션 정지.
- **모바일 세로/가로 전환**: `resize` 이벤트 시 현재 position 유지하며 크기만 재조정.
- **블로그 글 없음**: 히어로 카드 영역에 "아직 작성된 글이 없습니다" 플레이스홀더.
